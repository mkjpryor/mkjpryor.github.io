Work Experience

Senior Scientific Software Engineer, Met Office, Wallingford, 2008 - present
  • Technical lead and community contact for JULES, a land-surface model written in Fortran that can be run either as a standalone model or as part of the Met Office Unified Model.
  • Committer to the FCM and Rose repositories. Both systems are open-source tools developed primarily by the Met Office - FCM is a code management and Fortran build system and Rose is a tool for managing and executing suites of model runs.
  • Designed JULES website and maintain the content.
  • Migrated JULES development to a source control system and developed coding standards and procedures for incorporating code from the scientific community.
  • Designed and implemented the JULES I/O and user interface in a modular and extensible way.
  • Designed and implemented parallel scheme for JULES using MPI.
  • Ported the JULES documentation from a single, large Word document to the Sphinx documentation system, allowing the generation of HTML and PDF documentation from the same source files.
  • Implemented a benchmarking suite (initially using R, later Python/Numpy/Scipy) for JULES that generates HTML reports with interactive graphs based on comparisons of model runs against known-good-output and observations.
  • Implemented a GUI and automated testing suite for JULES.
  • Technical lead on an international project to include the Australian land-surface model and JULES under a common coupling framework.
  • Member of the JULES steering committee and UM Project board.
  • Co-supervised internal secondment project aimed at increasing the throughput of JULES, using both traditional profiling and by investigating the use of co-processors (Intel Xeon Phi) and GPUs.
  • Designed and delivered "Introduction to JULES" training at JULES conference.
